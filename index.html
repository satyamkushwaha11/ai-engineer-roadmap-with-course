Chapter 1: What is an LLM?

Before writing a single line of code, you need to understand what a Large Language Model actually is and how it works under the hood. This is the foundation everything else builds on.

📖 Theory ⏱ 25 min read 🔰 Beginner

What is a Large Language Model?

A Large Language Model (LLM) is a type of AI that has been trained on billions of text documents — books, websites, code, articles — to understand and generate human language.

Think of it like this: if you read every book ever written, every website on the internet, and every piece of code on GitHub, you would start to understand patterns in language. You'd know that after "The capital of France is..." the word "Paris" is very likely to follow. An LLM does exactly this — but at a massive scale with hundreds of billions of examples.

💡 Simple Definition An LLM is a program that predicts the most likely next word (or token) given the context of previous words — over and over — until it produces a complete, meaningful response.

Tokens — The Building Blocks

LLMs don't read text character by character or word by word. They read in tokens — small chunks of text that can be a word, part of a word, or punctuation.

🔑 Key Rule On average: 1 token ≈ 4 characters ≈ 0.75 words. So 100 words ≈ 133 tokens. This matters because AI APIs charge per token.

Why Tokens Matter for Developers

  • Cost: APIs charge per 1,000 tokens (input + output)
  • Context window: The maximum tokens a model can "remember" in one conversation
  • Speed: More tokens = slower response

GPT-4o128,000 tokens~96,000 words / 320 pages

The model doesn't "think" and then write a full answer. It generates one token at a time, each time predicting the most likely next token given everything before it. That's why responses stream word by word in ChatGPT or Claude.

import Anthropic from '@anthropic-ai/sdk';
import 'dotenv/config';

// Initialize the Claude client
const client = new Anthropic({
  apiKey: process.env.ANTHROPIC_API_KEY,
});

async function askClaude(question) {
  const response = await client.messages.create({
    model: 'claude-sonnet-4-6',   // model to use
    max_tokens: 1024,              // max response length
    messages: [
      {
        role: 'user',
        content: question
      }
    ]
  });

  // Extract the text from the response
  return response.content[0].text;
}

// Run it

Chapter 3: Prompt Engineering

Prompt engineering is the art of talking to AI effectively. A well-crafted prompt can make the difference between a useless response and a perfect one. This is your highest-leverage skill as an AI developer.

✍️ Technique ⏱ 40 min ⭐ Essential

Why Prompt Engineering Matters

The same AI model can give you completely different quality outputs based purely on how you phrase your request. Compare these two prompts sent to the same model:

Bad Prompt vs Good Prompt — Same Model, Wildly Different Results
❌ WEAK PROMPT

"Write code for login"

Result: Generic, incomplete, wrong language, no error handling
✅ STRONG PROMPT

"Write a Node.js Express login endpoint using JWT. Include input validation with zod, bcrypt password checking, and return proper error messages for invalid credentials."

Result: Production-ready code exactly as needed

Technique 1 — System Prompts (Role Prompting)

The system prompt defines the AI's identity, expertise, and rules. It's read before every user message. This is the most powerful single technique.

🔑 Formula for a Great System Prompt Role + Expertise + Rules + Output Format
System prompt examplesjavascript
// ❌ Weak system prompt
const weak = "You are a helpful assistant.";

// ✅ Strong system prompt — Code Reviewer
const codeReviewer = `
You are a senior JavaScript engineer with 10+ years of experience.
When reviewing code you:
- Check for security vulnerabilities (XSS, injection, auth issues)
- Identify performance problems
- Suggest modern ES2024+ patterns
- Point out missing error handling
Always structure your response as:
1. CRITICAL issues (must fix)
2. WARNINGS (should fix)
3. SUGGESTIONS (nice to have)
Keep feedback concise and actionable.
`;

// ✅ Strong system prompt — Customer Support Bot
const supportBot = `
You are a friendly customer support agent for TechStore.
Company policy:
- Offer refunds within 30 days, no questions asked
- Never make promises we cannot keep
- Escalate to human if customer is angry (say ESCALATE)
- Always end with: "Is there anything else I can help you with?"
Tone: Professional but warm. Never use jargon.
`;

const response = await client.messages.create({
  model: 'claude-sonnet-4-6',
  max_tokens: 1024,
  system: codeReviewer,  // 👈 inject your persona here
  messages: [{ role: 'user', content: 'Review this: const pwd = req.body.password' }]
});

Technique 2 — Few-Shot Prompting

Show the AI examples of exactly what you want before asking your real question. This is like showing an employee a sample of the work before asking them to do it.

Few-shot promptingjavascript
const response = await client.messages.create({
  model: 'claude-sonnet-4-6',
  max_tokens: 512,
  messages: [
    {
      role: 'user',
      content: `Extract the product name and price from these sentences.
Return as JSON.

Example 1:
Input: "I bought a Sony WH-1000XM5 for $299"
Output: {"product": "Sony WH-1000XM5", "price": 299}

Example 2:
Input: "The MacBook Air M3 costs $1099"
Output: {"product": "MacBook Air M3", "price": 1099}

Now do this one:
Input: "I got the Samsung Galaxy S24 Ultra for $1199"
Output:`
    }
  ]
});
// Claude follows the exact pattern: {"product": "Samsung Galaxy S24 Ultra", "price": 1199}
💡 When to use few-shot Use it when you need consistent output formatting — JSON extraction, classification, data transformation. 2–3 examples are usually enough.

Technique 3 — Chain of Thought (CoT)

For complex reasoning tasks, ask the AI to think step by step before giving an answer. This dramatically improves accuracy on math, logic, and multi-step problems.

Chain of thoughtjavascript
// ❌ Without chain of thought — may get wrong answer
const bad = "A user has 3 subscriptions at $9.99/month. They want to upgrade 2 of them to $19.99/month. What's their new monthly total?";

// ✅ With chain of thought — forces step-by-step reasoning
const good = `A user has 3 subscriptions at $9.99/month.
They want to upgrade 2 of them to $19.99/month.
What's their new monthly total?

Think through this step by step before giving the final answer.`;

// Claude will now reason:
// Step 1: Current cost = 3 × $9.99 = $29.97
// Step 2: Upgrading 2: 2 × $19.99 = $39.98
// Step 3: Keeping 1: 1 × $9.99 = $9.99
// Step 4: Total = $39.98 + $9.99 = $49.97
// Answer: $49.97/month

Technique 4 — Output Formatting

Always tell the AI exactly what format you want. If you're building an app, you need predictable, parseable output — not free-form text.

Technique 6 — Context Stuffing (RAG Preview)

LLMs only know what's in their training data. But you can give them any information by including it directly in the prompt. This is the foundation of RAG (we'll build it in Chapter 7).

Context stuffingjavascript
// Give Claude information it doesn't have in training data
const companyDocs = `
ACME Corp Refund Policy (Updated Jan 2026):
- Electronics: 30-day return, must be unopened
- Clothing: 60-day return, tags must be attached
- Digital products: No refunds after download
- Shipping costs are non-refundable
`;

const userQuestion = "Can I return headphones I opened 2 weeks ago?";

const response = await client.messages.create({
  model: 'claude-sonnet-4-6',
  max_tokens: 512,
  system: 'You are a customer support agent. Answer ONLY based on the policy provided. If the answer is not in the policy, say so.',
  messages: [{
    role: 'user',
    content: `Company Policy:
${companyDocs}

Customer Question: ${userQuestion}`

Multi-Turn Conversations

In a real chatbot, users send multiple messages. The AI needs conversation history to remember what was said earlier. LLMs are stateless — they don't remember anything between calls. You must send the full history every time.

How Conversation History Works
Turn 1 You send: messages: [{role:"user", content:"What is Node.js?"}]
Reply 1 Claude: "Node.js is a JavaScript runtime..."
Turn 2 You send ALL history: [user: "What is Node.js?", assistant: "Node.js is...", user: "How do I install it?"]
↑ The entire conversation is sent each time — the model has no memory of its own
Multi-turn chatbotjavascript
import Anthropic from '@anthropic-ai/sdk';
import 'dotenv/config';
import readline from 'readline';

const client = new Anthropic();

// This array stores the entire conversation history
const conversationHistory = [];

async function chat(userMessage) {
  // Add user message to history
  conversationHistory.push({
    role: 'user',
    content: userMessage
  });

  // Send the FULL history to Claude
  const response = await client.messages.create({
    model: 'claude-sonnet-4-6',
    max_tokens: 1024,
    system: 'You are a helpful JavaScript tutor. Be concise and use code examples.',
    messages: conversationHistory  // send everything
  });

  const assistantMessage = response.content[0].text;

  // Add Claude's reply to history too
  conversationHistory.push({
    role: 'assistant',
    content: assistantMessage
  });

  return assistantMessage;
}

// CLI chat loop
const rl = readline.createInterface({ input: process.stdin, output: process.stdout });

function askQuestion() {
  rl.question('\nYou: ', async (input) => {
    if (input.toLowerCase() === 'exit') { rl.close(); return; }
    const reply = await chat(input);
    console.log('\nClaude:', reply);
    askQuestion(); // loop
  });
}

console.log('Chat with Claude (type "exit" to quit)\n');
askQuestion();

Streaming Responses

Without streaming, the API waits for the full response before sending anything — your UI freezes for 5–10 seconds. With streaming, text appears word by word as it's generated, just like ChatGPT. This is always better for user experience.

Streaming with Claudejavascript
import Anthropic from '@anthropic-ai/sdk';
const client = new Anthropic();

async function streamResponse(question) {
  process.stdout.write('Claude: ');

  // Use stream() instead of create()
  const stream = await client.messages.stream({
    model: 'claude-sonnet-4-6',
    max_tokens: 1024,
    messages: [{ role: 'user', content: question }]
  });

  // Print each chunk as it arrives
  for await (const chunk of stream) {
    if (chunk.type === 'content_block_delta') {
      process.stdout.write(chunk.delta.text); // no newline — streams inline
    }
  }

  // Get the final complete message
  const finalMessage = await stream.finalMessage();
  console.log('\n\n--- Stream complete ---');
  console.log('Total tokens used:', finalMessage.usage.input_tokens + finalMessage.usage.output_tokens);
}

await streamResponse('Explain async/await in JavaScript with an example');

Streaming in a Next.js API Route

This is how you stream AI responses from a Next.js backend to your frontend:

app/api/chat/route.js (Next.js)javascript
import Anthropic from '@anthropic-ai/sdk';

const client = new Anthropic();

export async function POST(req) {
  const { messages } = await req.json();

  // Create a readable stream for the response
  const stream = new ReadableStream({
    async start(controller) {
      const encoder = new TextEncoder();

      const anthropicStream = await client.messages.stream({
        model: 'claude-sonnet-4-6',
        max_tokens: 2048,
        system: 'You are a helpful assistant.',
        messages: messages
      });

      for await (const chunk of anthropicStream) {
        if (chunk.type === 'content_block_delta') {
          // Send each token to the frontend
          controller.enqueue(encoder.encode(chunk.delta.text));
        }
      }

      controller.close();
    }
  });

  return new Response(stream, {
    headers: {
      'Content-Type': 'text/plain; charset=utf-8',
      'Transfer-Encoding': 'chunked'
    }
  });
}
Frontend — reading the stream (React)javascript
'use client';
import { useState } from 'react';

export default function ChatPage() {
  const [messages, setMessages] = useState([]);
  const [input, setInput] = useState('');
  const [response, setResponse] = useState('');

  async function sendMessage() {
    const userMsg = { role: 'user', content: input };
    const newMessages = [...messages, userMsg];
    setMessages(newMessages);
    setInput('');
    setResponse(''); // clear previous

    const res = await fetch('/api/chat', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ messages: newMessages })
    });

    // Read the stream
    const reader = res.body.getReader();
    const decoder = new TextDecoder();
    let fullText = '';

    while (true) {
      const { done, value } = await reader.read();
      if (done) break;
      fullText += decoder.decode(value);
      setResponse(fullText); // update UI as text arrives
    }

    // Add complete response to history
    setMessages(prev => [...prev, { role: 'assistant', content: fullText }]);
  }

  return (
    <div>
      {messages.map((m, i) => (
        <div key={i}><strong>{m.role}:</strong> {m.content}</div>
      ))}
      {response && <div><strong>assistant:</strong> {response}</div>}
      <input value={input} onChange={e => setInput(e.target.value)} />
      <button onClick={sendMessage}>Send</button>
    </div>
  );
}

Error Handling & Rate Limits

In production you must handle errors gracefully. APIs can fail due to rate limits, network issues, or invalid requests.

Production-ready error handlingjavascript
import Anthropic from '@anthropic-ai/sdk';
const client = new Anthropic();

async function safeAskClaude(question, retries = 3) {
  for (let attempt = 1; attempt <= retries; attempt++) {
    try {
      const response = await client.messages.create({
        model: 'claude-sonnet-4-6',
        max_tokens: 1024,
        messages: [{ role: 'user', content: question }]
      });
      return response.content[0].text;

    } catch (error) {
      // Rate limit — wait and retry
      if (error.status === 429) {
        const waitTime = Math.pow(2, attempt) * 1000; // exponential backoff
        console.log(`Rate limited. Waiting ${waitTime}ms before retry ${attempt}/${retries}`);
        await new Promise(r => setTimeout(r, waitTime));
        continue;
      }

      // Auth error — don't retry, fix the key
      if (error.status === 401) {
        throw new Error('Invalid API key. Check your ANTHROPIC_API_KEY environment variable.');
      }

      // Context too long
      if (error.status === 400) {
        throw new Error('Request too long. Reduce the message length.');
      }

      // Server error — retry
      if (error.status >= 500) {
        console.log(`Server error ${error.status}. Retry ${attempt}/${retries}`);
        await new Promise(r => setTimeout(r, 2000));
        continue;
      }

      throw error; // unknown error — re-throw
    }
  }
  throw new Error('Max retries exceeded');
}

Chapter 5: Embeddings Explained

Embeddings are the secret engine behind semantic search, RAG pipelines, and recommendation systems. Once you understand them, a whole new class of AI applications becomes possible.

🧮 Concept ⏱ 35 min ⭐ Core Skill

What is an Embedding?

An embedding is a way to convert any text (a word, sentence, or whole document) into a list of numbers — called a vector. These numbers capture the meaning of the text, not just its characters.

The key insight: texts with similar meaning get similar numbers. This lets computers measure how semantically related two pieces of text are — something impossible with traditional string matching.

Text → Embedding Vector (simplified to 4 dimensions)
"I love dogs"
→
[0.82, -0.14, 0.67, 0.33, ...]
"I adore cats"
→
[0.79, -0.11, 0.71, 0.28, ...]
"Stock market crash"
→
[-0.21, 0.88, -0.44, 0.91, ...]
"I love dogs" and "I adore cats" have similar vectors (close meaning) → "Stock market crash" has very different vector (different topic)
💡 Real embedding vectors are 1536 numbers long OpenAI's text-embedding-3-small produces 1536-dimensional vectors. Each dimension captures some aspect of meaning. You never read these numbers directly — you just measure distance between them.

Why Embeddings Matter — Semantic Search

Traditional search looks for exact keyword matches. Semantic search using embeddings finds results by meaning — even if the words are completely different.

Keyword Search vs Semantic Search
❌ KEYWORD SEARCH

Query: "car won't start"

Finds: documents containing the exact words "car", "won't", "start"

Misses: "vehicle fails to ignite", "automobile engine dead"

✅ SEMANTIC SEARCH

Query: "car won't start"

Finds: all conceptually similar documents

Also finds: "vehicle won't turn on", "engine not starting", "battery dead"

Cosine Similarity — How We Measure "Closeness"

We measure how similar two embeddings are using cosine similarity. It returns a value from -1 to 1:

ScoreMeaningExample
0.9 – 1.0Nearly identical meaning"dog" vs "puppy"
0.6 – 0.9Related topic"dog" vs "pet"
0.3 – 0.6Loosely related"dog" vs "animal"
0.0 – 0.3Unrelated"dog" vs "stock market"
Cosine similarity from scratchjavascript
// Cosine similarity — measures angle between two vectors
function cosineSimilarity(a, b) {
  const dotProduct = a.reduce((sum, val, i) => sum + val * b[i], 0);
  const magnitudeA  = Math.sqrt(a.reduce((sum, val) => sum + val * val, 0));
  const magnitudeB  = Math.sqrt(b.reduce((sum, val) => sum + val * val, 0));
  return dotProduct / (magnitudeA * magnitudeB);
}

// Example
const dogVector = [0.82, 0.14, 0.67, 0.33];
const catVector = [0.79, 0.11, 0.71, 0.28];
const bankVector = [-0.21, 0.88, -0.44, 0.91];

console.log(cosineSimilarity(dogVector, catVector));  // ~0.99 (very similar!)
console.log(cosineSimilarity(dogVector, bankVector)); // ~0.12 (unrelated)

async function buildIndex(docs) {

  const embeddings = await openai.embeddings.create({
    model: 'text-embedding-3-small',
    input: texts
  });

    ...doc,
    vector: embeddings.data[i].embedding

// Step 2: Cosine similarity
function cosineSimilarity(a, b) {
  const dot = a.reduce((s, v, i) => s + v * b[i], 0);
  const magA = Math.sqrt(a.reduce((s, v) => s + v * v, 0));
  const magB = Math.sqrt(b.reduce((s, v) => s + v * v, 0));
  return dot / (magA * magB);
}

// Step 3: Search — embed the query, find most similar docs
async function search(query, index, topK = 3) {
  const queryEmbed = await openai.embeddings.create({
    model: 'text-embedding-3-small',
    input: query

  // Score every document
  const results = index.map(doc => ({
    ...doc,
    score: cosineSimilarity(queryVector, doc.vector)
  }));

  // Return top K results sorted by score
  return results
    .sort((a, b) => b.score - a.score)
    .slice(0, topK);
}

// Simple chunker — splits by word count with overlap
function chunkText(text, chunkSize = 500, overlap = 50) {
  const words = text.split(/\s+/);
  const chunks = [];

  for (let i = 0; i < words.length; i += chunkSize - overlap) {
    const chunk = words.slice(i, i + chunkSize).join(' ');
    if (chunk.trim()) {
      chunks.push({
        text: chunk,
        startIndex: i,
        // Keep metadata so you can find the source later
        metadata: { wordStart: i, wordEnd: i + chunkSize }
      });
    }
  }
  return chunks;
}

import fs from 'fs';
import path from 'path';

// Load text from a file
export function loadDocument(filePath) {
  const ext = path.extname(filePath).toLowerCase();
  if (ext === '.txt' || ext === '.md') {
    return fs.readFileSync(filePath, 'utf-8');
  }
  throw new Error(`Unsupported file type: ${ext}. Use .txt or .md`);
}

// Split text into overlapping chunks
export function chunkDocument(text, options = {}) {
  const {
    chunkSize = 500,     // words per chunk
    overlap = 50,        // words of overlap between chunks
    source = 'unknown'  // metadata: where did this come from?
  } = options;

  const words = text.split(/\s+/).filter(Boolean);
  const chunks = [];

  for (let i = 0; i < words.length; i += chunkSize - overlap) {
    const chunkWords = words.slice(i, i + chunkSize);
    if (chunkWords.length < 20) break; // skip tiny tail chunks

    chunks.push({
      id: `${source}-chunk-${chunks.length}`,
      text: chunkWords.join(' '),
      metadata: {
        source,
        chunkIndex: chunks.length,
        wordStart: i,
        wordEnd: i + chunkWords.length
      }
    });
  }

  console.log(`✅ Created ${chunks.length} chunks from "${source}"`);
  return chunks;
}

    });
    allVectors.push(...response.data.map(d => d.embedding));
    console.log(`  Embedded ${Math.min(i + BATCH_SIZE, chunks.length)}/${chunks.length} chunks`);
  }

  return allVectors;
}

// Index all chunks into Qdrant
export async function indexChunks(chunks) {
  console.log(`\nIndexing ${chunks.length} chunks...`);
  const vectors = await embedInBatches(chunks);

  const points = chunks.map((chunk, i) => ({
    id: i + 1,
    vector: vectors[i],
    payload: {
      text: chunk.text,
      source: chunk.metadata.source,
      chunkIndex: chunk.metadata.chunkIndex
    }
  }));

  // Upsert all at once
  await qdrant.upsert(COLLECTION, { points, wait: true });
  console.log(`✅ Indexed ${points.length} chunks successfully`);
}

  {
    name: 'get_current_weather',
    description: 'Get the current weather for a city',
    input_schema: {
      type: 'object',
      properties: {
        city: { type: 'string', description: 'City name, e.g. London' }
      },
      required: ['city']
    }
  },
  {
    name: 'calculate',
    description: 'Perform a mathematical calculation',
    input_schema: {
      type: 'object',
      properties: {
        expression: { type: 'string', description: 'Math expression, e.g. 15 * 24 + 100' }
      },
      required: ['expression']
    }
  }
];

// ── Actual tool implementations ──
async function executeTool(name, input) {
  if (name === 'get_current_weather') {
    // In real code: call a weather API
    return { city: input.city, temp: '22°C', condition: 'Sunny', humidity: '45%' };
  }
  if (name === 'calculate') {
    // Safe eval using Function constructor
    const result = new Function(`return ${input.expression}`)();
    return { result, expression: input.expression };
  }
  throw new Error(`Unknown tool: ${name}`);
}

// ── The agent loop ──
async function runAgent(userMessage) {
  console.log(`\nUser: ${userMessage}\n`);
  const messages = [{ role: 'user', content: userMessage }];

  for (let step = 0; step < 10; step++) { // max 10 steps to prevent infinite loops
    const response = await claude.messages.create({
      model: 'claude-sonnet-4-6',
      max_tokens: 1024,
      tools,
      messages,
      system: 'You are a helpful assistant. Use tools when needed to answer accurately.'
    });

    console.log(`Step ${step + 1}: stop_reason = ${response.stop_reason}`);

    // Agent is done — has a final text answer
    if (response.stop_reason === 'end_turn') {
      const finalAnswer = response.content.find(b => b.type === 'text')?.text;
      console.log(`\nAgent: ${finalAnswer}`);
      return finalAnswer;
    }

    // Agent wants to use a tool
    if (response.stop_reason === 'tool_use') {
      // Add the agent's response to history
      messages.push({ role: 'assistant', content: response.content });

      // Execute each tool it requested
      const toolResults = [];
      for (const block of response.content) {
        if (block.type === 'tool_use') {
          console.log(`  🔧 Using tool: ${block.name}(${JSON.stringify(block.input)})`);
          const result = await executeTool(block.name, block.input);
          console.log(`  ✅ Result: ${JSON.stringify(result)}`);

          toolResults.push({
            type: 'tool_result',
            tool_use_id: block.id,
            content: JSON.stringify(result)
          });
        }
      }

      // Add tool results to history and loop
      messages.push({ role: 'user', content: toolResults });
    }
  }
  throw new Error('Agent exceeded maximum steps');
}

Chapter 10: Tool Calling (Function Calling)

Tool calling is the mechanism that gives LLMs superpowers — the ability to interact with APIs, run code, search the web, and take real actions in the world. Master this and you can build almost any AI application.

🔧 Tools ⏱ 50 min ⭐ Core Skill

How Tool Calling Works

When you define tools for an LLM, you're telling it: "These actions exist and here's how to use them." The model then decides when to call them based on the user's request.

Tool Calling Flow
1. You Send message + tool definitions (JSON schema) to the LLM
2. LLM Decides which tool to call and with what arguments → returns tool_use block
3. You Execute the tool with those arguments in YOUR code → get result
4. You Send the result back to the LLM as a tool_result
5. LLM Uses the result to form the final answer — or calls another tool
The LLM never directly calls the tool — it only tells YOU what to call. You execute it and report back.

Chapter 11: ReAct Pattern & Planning

ReAct (Reasoning + Acting) is the foundational pattern behind every AI agent. It makes agents transparent, debuggable, and dramatically more accurate than agents that act without thinking first.

🧠 Pattern⏱ 40 min⭐ Core Concept

What is ReAct?

ReAct stands for Reason + Act. Instead of immediately calling a tool, the agent first thinks out loud about what to do, then acts. This "thinking step" makes agents far more accurate and easier to debug.

ReAct Loop — Each Step Has Thought → Action → Observation
Thought 1: The user wants to know the cheapest flight to Paris. I need to search for flights first.
Action 1: search_flights({destination: "Paris", date: "2026-07-15"})
Observation 1: Found 3 flights: $450 (Air France), $380 (EasyJet), $510 (British Airways)
Thought 2: EasyJet at $380 is cheapest. I should also check if the user wants economy or business class before finalizing.
Final Answer: The cheapest flight to Paris on July 15 is EasyJet at $380. Would you like economy or business?

class SmartMemoryChat {
  constructor() {
    this.messages  = [];

async function recallMemories(userId, query, limit = 5) {
  const embed = await openai.embeddings.create({
    model: 'text-embedding-3-small', input: query

  const results = await qdrant.search('agent_memories', {
    vector: embed.data[0].embedding,
    limit,
    with_payload: true,
    filter: { must: [{ key: 'userId', match: { value: userId } }] },
    score_threshold: 0.5

}

          if (content.toLowerCase().includes(args.query.toLowerCase())) {
            const lineNum = content.split('\n').findIndex(l => l.toLowerCase().includes(args.query.toLowerCase())) + 1;
            matches.push(`${file}:${lineNum}`);
          }
        }
      }
      return { content: [{ type: 'text', text: matches.length ? matches.join('\n') : 'No matches found' }] };
    }

    throw new Error(`Unknown tool: ${name}`);

  } catch (error) {
    return { content: [{ type: 'text', text: `Error: ${error.message}` }], isError: true };
  }
});

// Start the server (communicates via stdio)
const transport = new StdioServerTransport();
await server.connect(transport);
console.error('MCP server running...'); // use stderr — stdout is for protocol

async function judgeAnswer(question, actualAnswer, expectedAnswer) {
  const res = await claude.messages.create({
    model: 'claude-haiku-4-5',  // cheap model for judging
    max_tokens: 256,
    system: 'You are an evaluation judge. Score AI responses objectively.',
    messages: [{
      role: 'user',
      content: `Question: ${question}
Expected: ${expectedAnswer}
Actual: ${actualAnswer}

Score the actual answer on these criteria (1-5 each):
- Correctness: Is the information accurate?
- Completeness: Does it address the question fully?
- Clarity: Is it easy to understand?

Return JSON: {"correctness": N, "completeness": N, "clarity": N, "overall": N, "pass": true/false}`
    }]
  });

  return JSON.parse(res.content[0].text);
}

// Run full eval suite
async function runEvals(answerFn) {
  console.log(`\n🧪 Running ${evalDataset.length} evaluations...\n`);
  const results = [];

  for (const item of evalDataset) {
    const answer = await answerFn(item.input);
    const score  = await judgeAnswer(item.input, answer, item.expected);

    results.push({ ...item, answer, score });
    const icon = score.pass ? '✅' : '❌';
    console.log(`${icon} "${item.input.slice(0, 40)}..." → overall: ${score.overall}/5`);
  }

  // Summary
  const passed    = results.filter(r => r.score.pass).length;
  const avgScore  = results.reduce((s, r) => s + r.score.overall, 0) / results.length;
  const passRate  = (passed / results.length * 100).toFixed(1);

  console.log(`\n📊 Results: ${passed}/${results.length} passed (${passRate}%)`);
  console.log(`📊 Avg score: ${avgScore.toFixed(2)}/5`);

  // Fail CI if quality drops below threshold
  if (parseFloat(passRate) < 80) {
    console.error('❌ EVAL FAILED: Pass rate below 80%');
    process.exit(1);
  }

  return results;
}

        });
        return res.choices[0].message.content;
      }

Fine-tuning customizes a model's behavior on your specific domain. Multimodal AI adds vision, audio, and other modalities beyond text. These are your senior-level differentiators as an AI engineer.

import OpenAI from 'openai';
import fs     from 'fs';

const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

async function runFineTuning() {
  // Step 1: Upload training file
  console.log('Uploading training data...');
  const file = await openai.files.create({
    file:    fs.createReadStream('./training-data.jsonl'),
    purpose: 'fine-tune'
  });
  console.log('File uploaded:', file.id);

  // Step 2: Start fine-tuning job
  const job = await openai.fineTuning.jobs.create({
    training_file: file.id,
    model:         'gpt-4o-mini-2024-07-18', // cheap but powerful base model
    hyperparameters: {
      n_epochs: 3  // number of training passes (3-4 is usually good)
    }
  });
  console.log('Training started. Job ID:', job.id);

  // Step 3: Poll for completion
  let status = job.status;
  while (status !== 'succeeded' && status !== 'failed') {
    await new Promise(r => setTimeout(r, 30000)); // check every 30s
    const updated = await openai.fineTuning.jobs.retrieve(job.id);
    status = updated.status;
    console.log(`Status: ${status} | Trained tokens: ${updated.trained_tokens || 0}`);
  }

  if (status === 'succeeded') {
    const completed = await openai.fineTuning.jobs.retrieve(job.id);
    console.log('✅ Fine-tuning complete!');
    console.log('Your model ID:', completed.fine_tuned_model);
    // e.g.: ft:gpt-4o-mini-2024-07-18:org:name:ABC123
    return completed.fine_tuned_model;
  }
}

// Step 4: Use your fine-tuned model
async function useFineTunedModel(modelId, text) {
  const res = await openai.chat.completions.create({
    model:    modelId,  // your fine-tuned model ID
    messages: [
      { role: 'system', content: 'You are a JSON extractor. Return only valid JSON.' },
      { role: 'user',   content: `Extract: ${text}` }
    ]
  });
  return JSON.parse(res.choices[0].message.content);
}

async function analyzeImage(imagePath, question) {
  const imageData   = fs.readFileSync(imagePath);
  const base64Image = imageData.toString('base64');
  const mimeType    = imagePath.endsWith('.png') ? 'image/png' : 'image/jpeg';

  const response = await claude.messages.create({
    model:      'claude-sonnet-4-6',
    max_tokens: 1024,
    messages: [{
      role: 'user',
      content: [
        {
          type:   'image',
          source: { type: 'base64', media_type: mimeType, data: base64Image }
        },
        { type: 'text', text: question }
      ]
    }]
  });

  return response.content[0].text;
}

import express  from 'express';
import multer   from 'multer';
import Anthropic from '@anthropic-ai/sdk';
import fs from 'fs';

const app    = express();
const upload = multer({ dest: 'uploads/' });
const claude = new Anthropic();

// POST /scan-receipt — upload an image, get structured data back
app.post('/scan-receipt', upload.single('receipt'), async (req, res) => {
  if (!req.file) return res.status(400).json({ error: 'No image uploaded' });

  const imageData   = fs.readFileSync(req.file.path);
  const base64Image = imageData.toString('base64');
  const mimeType    = req.file.mimetype;

  const response = await claude.messages.create({
    model:      'claude-sonnet-4-6',
    max_tokens: 1024,
    messages: [{
      role: 'user',
      content: [
        { type: 'image', source: { type: 'base64', media_type: mimeType, data: base64Image } },
        {
          type: 'text',
          text: `Extract all data from this receipt. Return ONLY valid JSON:
{
  "store": "store name",
  "date": "YYYY-MM-DD",
  "items": [{"name": "item", "price": 0.00, "quantity": 1}],
  "subtotal": 0.00,
  "tax": 0.00,
  "total": 0.00,
  "paymentMethod": "card type or cash"
}`
        }
      ]
    }]
  });

  // Clean up uploaded file
  fs.unlinkSync(req.file.path);

  try {
    const data = JSON.parse(response.content[0].text);
    res.json({ success: true, data });
  } catch {
    res.json({ success: false, raw: response.content[0].text });
  }
});

app.listen(3000, () => console.log('Receipt scanner on :3000'));
